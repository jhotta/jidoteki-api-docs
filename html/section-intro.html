The Jidoteki beta API is a REST API which allows you to offload and automate your operating system installations. The API is still in beta, so this documentation will continue to be updated as new features are added.

Getting started

Below you'll find some information and common keywords found throughout this documentation.

Keyword Description
JSON All results are returned as JSON strings.
HTTPS For security purposes, all API requests must be made over HTTPS.
TIMESTAMPS All timestamps are returned as UNIX timestamps in UTC, with 6 digit precision (microseconds).
Example: 1363414483.832038
CONTENT-TYPE The content-type for all requests is a standard JSON MIME media type.
Example: application/json
PARAMETERS Request parameters must be sent as a JSON-formatted string.
Example: {"os_id":"debian-7.1.0"}
HMAC Each API request must contain an HMAC (signature) of the entire request. Details are in the Authentication section.
API KEY A user's secret API KEY is used to generate each HMAC signature.
ACCESS TOKENS An access token is a temporary unique identification string sent as an HTTP header. It expires after 8 hours.
USER-AGENT You can use almost any HTTP header User-Agent you want, limited to 128 characters.
Example: curl/7.21.4 (macbook air @ home)
IPv6 The API is accessible over IPv4 and IPv6.
Example: curl --ipv6
SHA-256 The hashing algorithm used to generate an HMAC signature is SHA-256.
UID When requesting an ACCESS TOKEN, you need to supply your UID (User ID) which is a 7-character alphanumeric string.
